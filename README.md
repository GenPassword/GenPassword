# GenPass — генератор паролей

**GenPass** — MVP веб-приложения для генерации надёжных паролей с настраиваемыми параметрами, оценкой стойкости и сохранением пользовательских пресетов.

| Ресурс | Ссылка | Доступность |
|--------|--------|-------------|
| **Frontend (GitHub Pages)** | [https://genpassword.github.io/GenPassword/](https://genpassword.github.io/GenPassword/) | Постоянно; API — через туннель (см. ниже) |
| **Frontend + Backend (сервер УрФУ)** | `http://10.40.241.58` | **Только из сети университета** (основной вариант для защиты) |
| **API (Cloudflare Tunnel)** | [https://myproject24.ru/api](https://myproject24.ru/api) | **Только при активном туннеле**; используется фронтом с GitHub Pages |

> Два **независимых контура**:
>
> 1. **Защита проекта (УрФУ):** на сервере УрФУ размещены **отдельно** фронтенд и бэкенд (+ PostgreSQL). Фронт на УрФУ обращается к API **на том же сервере** (`10.40.241.58`). Весь контур доступен **только из сети университета**.
>
> 2. **Публичная демо-версия (GitHub):** статический фронт на **GitHub Pages** обращается к API через **Cloudflare Tunnel** (`myproject24.ru`). Туннель пробрасывает запросы на бэкенд УрФУ; работает **только пока туннель запущен** — это не круглосуточный хостинг.

## Описание продукта

MVP веб-приложения с **двумя вариантами размещения**:

- **Сервер УрФУ** — фронтенд и бэкенд (ASP.NET Core + PostgreSQL) размещены отдельно на университетском сервере; фронт общается с API напрямую. Доступ **из сети УрФУ** (защита проекта).
- **GitHub Pages** — публичный статический фронт; запросы к API идут через **Cloudflare Tunnel** на `myproject24.ru` (пока туннель активен).

Приложение предоставляет пользователю возможность:

1. **Генерировать пароль** по настройкам: длина, наборы символов (строчные / прописные / цифры / спецсимволы), опция «исключить похожие символы» (`O`, `0`, `I`, `l`, `1`).
2. **Проверять качество результата** — индикатор «слабый / средний / надёжный / очень надёжный» на основе энтропии и правил валидации, кнопка «Копировать».
3. **Сохранять удобные настройки (пресеты)** и быстро применять их повторно (для авторизованных пользователей — в PostgreSQL на сервере УрФУ; тема интерфейса хранится в `localStorage`).

Дополнительно реализованы режимы PIN-кода и пароля из слов, авторизация пользователей, сохранение сгенерированных паролей с шифрованием и тёмная тема.

## Функциональность

### Генерация паролей

| Режим | Описание |
|-------|----------|
| **Случайный** | Криптографически стойкая генерация (`RandomNumberGenerator`) с настройкой длины (4–64), наборов символов, минимального количества цифр и спецсимволов, исключения похожих символов и запрета повторов |
| **PIN-код** | Числовой код длиной 4–8 цифр, опционально без повторяющихся цифр |
| **Пароль из слов** | Passphrase из словаря: 2–8 слов, выбор регистра и разделителя (`-`, `_` или без разделителя) |

### Оценка и UX

- Визуальный индикатор стойкости (полоска + текст + процент)
- Копирование пароля одним кликом (кнопка или клик по блоку пароля)
- Светлая и тёмная тема (сохраняется в `localStorage`)
- Адаптивный интерфейс

### Персонализация (требует регистрации)

- **Пресеты настроек** — сохранение, применение и удаление конфигураций для каждого режима генерации
- **Мои пароли** — сохранение сгенерированных паролей с описанием (шифрование на сервере)
- JWT-авторизация с refresh-токеном в HttpOnly-cookie

### Безопасность

- CSP-заголовки на фронтенде
- Санитизация пользовательского ввода (защита от XSS)
- Rate limiting на API
- Проверка сгенерированных паролей на последовательности и повторы (`SequentialRule`, `RepetitionRule`)
- Логирование (NLog) и трассировка (OpenTelemetry)

## Локальный запуск

Раздел для **разработки** на своей машине. Для защиты проекта сервер работает на **сервере УрФУ** (см. [Деплой](#деплой)).

### Требования

- [.NET 8 SDK](https://dotnet.microsoft.com/download/dotnet/8.0)
- [Node.js](https://nodejs.org/) 18+ и npm
- Подключение к **сети УрФУ** — если локальный бэкенд должен ходить в PostgreSQL на `10.40.241.58` (как на сервере)

> Локальный `dotnet run` не заменяет хостинг на УрФУ: для защиты используется приложение, развёрнутое на университетском сервере.

### 1. Бэкенд

```bash
cd backend/PasswordGenerator/PasswordGenerator
```

Настройте строку подключения к PostgreSQL в `appsettings.json` или через переменные окружения (см. [Конфигурация](#конфигурация)). По умолчанию указан хост сервера УрФУ (`10.40.241.58`); подключение работает **только из сети университета**.

```bash
# Восстановление зависимостей и запуск
dotnet restore
dotnet run
```

API будет доступен по адресу `https://localhost:7xxx` или `http://localhost:5xxx` (порт указан в консоли при запуске).

Миграции базы данных применяются автоматически при старте приложения.

### 2. Frontend

```bash
cd passwordGenerated
npm install
npm run dev
```

Dev-сервер Vite: `http://localhost:5173`

> **Важно:** для локальной разработки измените URL API в `passwordGenerated/src/main.js` на адрес локального бэкенда (например, `http://localhost:5000/api/...`) и убедитесь, что origin добавлен в CORS-политику в `Program.cs`.

### 3. Быстрая проверка генерации (без фронтенда)

```bash
curl -X POST http://localhost:5000/api/password/generate \
  -H "Content-Type: application/json" \
  -d '{
    "length": 16,
    "includeLowercase": true,
    "includeUppercase": true,
    "includeDigits": true,
    "includeSpecial": true,
    "excludeSimilar": true,
    "noRepeats": false,
    "minDigits": 2,
    "minSpecial": 1
  }'
```

Пример ответа:

```json
{
  "password": "Kx#9mP2nQw5rTv8y",
  "entropy": 95.27,
  "strengthLevel": 2
}
```

## Настройки генерации

### Случайный пароль

| Параметр | Тип | Диапазон / значения | Описание |
|----------|-----|---------------------|----------|
| `length` | int | 4–64 | Длина пароля |
| `includeLowercase` | bool | — | Строчные буквы `a-z` |
| `includeUppercase` | bool | — | Прописные буквы `A-Z` |
| `includeDigits` | bool | — | Цифры `0-9` |
| `includeSpecial` | bool | — | Спецсимволы `!@#$%^&*...` |
| `excludeSimilar` | bool | — | Исключить `O`, `0`, `I`, `l`, `1` |
| `noRepeats` | bool | — | Запретить повторяющиеся символы |
| `minDigits` | int | 0–length | Минимальное количество цифр |
| `minSpecial` | int | 0–length | Минимальное количество спецсимволов |

**Ограничения валидации:**

- Должна быть выбрана хотя бы одна группа символов
- `minDigits + minSpecial ≤ length`
- При `noRepeats: true` длина не может превышать размер итогового алфавита

### PIN-код

| Параметр | Диапазон | Описание |
|----------|----------|----------|
| `length` | 4–8 | Длина PIN |
| `noRepeats` | bool | Без повторяющихся цифр |

### Пароль из слов

| Параметр | Диапазон / значения | Описание |
|----------|----------------------|----------|
| `wordCount` | 2–8 | Количество слов |
| `wordCase` | `0` / `1` / `2` | Нижний / ВЕРХНИЙ / С заглавной |
| `separator` | `-`, `_`, `""` | Разделитель между словами |

## Оценка надёжности пароля

Стойкость рассчитывается на **бэкенде** по формуле энтропии:

```
entropy = length × log₂(alphabetSize)
```

где `alphabetSize` — размер используемого алфавита с учётом выбранных наборов символов и опции `excludeSimilar`.

### Уровни стойкости (бэкенд)

| Энтропия (бит) | Уровень |
|----------------|---------|
| < 28 | VeryWeak |
| 28 – 35 | Weak |
| 36 – 59 | Fair |
| 60 – 127 | Strong |
| ≥ 128 | VeryStrong |

### Отображение на фронтенде

| Энтропия | Метка |
|----------|-------|
| < 28 | Слабый |
| 28 – 35 | Средний |
| 36 – 59 | Надёжный |
| ≥ 60 | Очень надёжный |

Процент надёжности: `min(100, round(entropy / 80 × 100))`.

Дополнительно пароль проверяется правилами: отсутствие длинных последовательностей (`SequentialRule`) и повторяющихся паттернов (`RepetitionRule`).

## REST API

Бэкенд один (сервер УрФУ); **URL зависит от того, какой фронтенд обращается**:

| Кто вызывает API | Базовый URL |
|------------------|-------------|
| Фронт на **сервере УрФУ** (из сети университета) | `http://10.40.241.58:<port>/api` |
| Фронт на **GitHub Pages** (через туннель) | `https://myproject24.ru/api` |
| Локальная разработка | `http://localhost:<port>/api` |

Порт ASP.NET Core задаётся конфигурацией на сервере УрФУ или в выводе `dotnet run`.

### Генерация (без авторизации)

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| `POST` | `/password/generate` | Случайный пароль / PIN |
| `POST` | `/password/generate-from-words` | Пароль из слов |

### Аутентификация

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| `POST` | `/Auth/register` | Регистрация `{ email, password }` |
| `POST` | `/Auth/login` | Вход, возвращает `{ token }` |
| `POST` | `/Auth/refresh` | Обновление access token (cookie) |
| `POST` | `/Auth/logout` | Выход, отзыв refresh token |

### Пресеты настроек (JWT)

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| `GET` | `/UserSettings/{type}` | Список пресетов (`Random`, `Pin`, `Words`) |
| `POST` | `/UserSettings/save` | Сохранить пресет |
| `DELETE` | `/UserSettings/delete` | Удалить пресет `{ Id }` |

### Сохранённые пароли (JWT)

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| `GET` | `/UserSavedPassword/getSaves` | Список сохранённых паролей |
| `POST` | `/UserSavedPassword/save` | Сохранить пароль |
| `DELETE` | `/UserSavedPassword/delete` | Удалить пароль `{ Id }` |

## Деплой

### Контур GitHub Pages + Cloudflare Tunnel

**Frontend** — публикуется на GitHub Pages (интерфейс доступен постоянно):

```bash
cd passwordGenerated
npm run deploy
```

В `main.js` должны быть URL туннеля (`https://myproject24.ru/api/...`) — как в текущей версии репозитория.

**API для GitHub-фронта** — бэкенд на УрФУ пробрасывается через [Cloudflare Tunnel](https://developers.cloudflare.com/cloudflare-one/connections/connect-networks/) на `myproject24.ru`. Туннель запускается вручную; пока он не активен, фронт на GitHub Pages не сможет генерировать пароли.

1. На сервере УрФУ API слушает локальный порт (например, `5000`).
2. Cloudflare Tunnel направляет `myproject24.ru` → `http://localhost:5000`.
3. В `Program.cs` в CORS указан `https://genpassword.github.io`.

---

### Контур сервера УрФУ (защита проекта)

**Backend + PostgreSQL** на университетском сервере (`10.40.241.58`):

```bash
cd backend/PasswordGenerator/PasswordGenerator
dotnet publish -c Release -o ./publish
```

1. Скопировать `publish/` на сервер УрФУ, задать переменные по `deploy.env.example`.
2. Запустить API (systemd, IIS или `dotnet PasswordGenerator.dll`).
3. PostgreSQL на том же сервере — строка подключения в `appsettings.json`.

**Frontend** — отдельно собрать и выложить статику на тот же сервер УрФУ:

```bash
cd passwordGenerated
# В main.js указать API: http://10.40.241.58:<port>/api/...
npm run build
# Содержимое dist/ — на веб-сервер УрФУ (nginx, IIS и т.д.)
```

Фронт на УрФУ доступен **только из сети университета**.
